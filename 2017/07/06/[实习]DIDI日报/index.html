为什么使用DOCKER：

快速搭建环境

开发模式：

1、Gitlab下载代码到本地电脑上，建立自己的分支，在分支上开发，编写完成后提交到Gitlab

2、登录DOCKER，Gitlab拉取最新代码，并进行build运行

3、测试接口功能

PHP框架：CodeIgniter

应用程序流程图：

1、index.php 文件作为前端控制器，初始化运行 CodeIgniter 所需的基本资源；
2、Router 检查 HTTP 请求，以确定如何处理该请求；
3、如果存在缓存文件，将直接输出到浏览器，不用走下面正常的系统流程；
4、在加载应用程序控制器之前，对 HTTP 请求以及任何用户提交的数据进行安全检查；
5、控制器加载模型、核心类库、辅助函数以及其他所有处理请求所需的资源；
6、最后一步，渲染视图并发送至浏览器，如果开启了缓存，视图被会先缓存起来用于 后续的请求。

CodeIgniter MVC：

  • 模型（models）
  • 视图（views）
  • 控制器（controllers）

接口访问形式：

即 

http://10.95.121.208:8000 映射到 /home/xiaoju/webroot/gulfstream

1、静态文件，位于根目录static下，根目录+相对地址确定URL，举个例子

http://10.95.121.208:8000/static/mis/webpack.config.js

2、动态路由

config/routes.php配置路由

$route['default_controller'] = "welcome";

$route['gulfstream/passenger/v2/(.+)'] = "$1";

Nginx 配置

server {
    listen       8000;
    root /home/xiaoju/webroot/gulfstream;

    location /gulfstream/api/static/
    {
        rewrite ^/gulfstream/api/static/(.*)$  /static/$1 last;
    }

    location  /gulfstream
    {
        if ($request_method = 'GET') {
            #add_header 'Access-Control-Allow-Origin' 'http://static.udache.com';
            add_header 'Access-Control-Allow-Credentials' '*';
            add_header 'Access-Control-Allow-Credentials' 'true';
            add_header 'Access-Control-Allow-Methods' 'GET, POST, OPTIONS';
            add_header 'Access-Control-Allow-Headers' 'Authorization,Accept,Keep-Alive,User-Agent,X-Requested-With,If-Modified-Since,Cache-Control,Content-Type';
        }
        if ($request_uri ~* (\'|\<|\"|\>|\.svn)) {
          return 403 ;
    }
    rewrite ^/gulfstream/([^\/]*)/([^\/]*)/(.*)$ /application/$1/$2/index.php/$3 break;
}

web根目录位于

/home/xiaoju/webroot/gulfstream

URL -> 物理路径映射

/gulfstream/(1)/(2)/(3) => /home/xiaoju/webroot/gulfstream/application/(1)/(2)/index.php/(3)

（1）表示模块，在applicatin目录下有多个模块

（2）表示版本，分别是v1、v2…

（3）为CI工程下controllers文件夹为根目录，URL由类名+方法名组成
